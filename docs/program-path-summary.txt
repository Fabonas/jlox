jlox Program Path Summary
==========================

A Lox program follows one of two entry paths, then passes through the same
four-phase pipeline.

ENTRY POINTS
------------
1. java com.lox.Lox script.lox
   -> Lox.main(args)
   -> Lox.runFile(path)
   -> Lox.run(source)

2. java com.lox.Lox
   -> Lox.main(args)
   -> Lox.runPrompt()
   -> (loop) Lox.run(source) for each line

com.lox.Main also exists as an alternate entry point; it simply delegates to
Lox.main(args).

PHASE 1: SCANNING
-----------------
Lox.run(source)
  -> new Scanner(source)
  -> Scanner.scanTokens()
  -> returns List<Token>

The scanner converts raw characters into tokens. It reports unexpected
characters through Lox.error(line, message).

PHASE 2: PARSING
----------------
  -> new Parser(tokens)
  -> Parser.parse()
  -> returns List<Stmt> (the AST)

The parser is recursive-descent. Syntax errors are reported through
Lox.error(token, message) and the parser synchronizes to the next statement
boundary.

PHASE 3: RESOLVING
------------------
  -> new Resolver(interpreter)
  -> Resolver.resolve(stmts)

The resolver walks the AST and records the lexical depth of every local
variable reference in the interpreter's locals map via resolve(expr, depth).
It also performs static checks:
  - variable read inside its own initializer
  - return outside a function
  - duplicate local variables
  - invalid use of this / super

PHASE 4: INTERPRETING
---------------------
  -> interpreter.interpret(stmts)
  -> execute(stmt) for each statement
  -> evaluate(expr) for expressions

The interpreter is a tree-walking evaluator. Runtime errors are thrown as
RuntimeError and caught by interpret() -> Lox.runtimeError(error).

Execution paths inside the interpreter:
  - Block     : new Environment(env) -> executeBlock()
  - Var       : evaluate initializer -> env.define()
  - Assign    : evaluate value -> env.assign() or env.assignAt()
  - Variable  : env.get() or env.getAt()
  - If/While  : evaluate condition -> execute branch/body
  - Function  : create LoxFunction closure -> env.define()
  - Call      : evaluate callee & args -> LoxCallable.call()
  - Class     : evaluate superclass -> define class -> bind methods
  - Return    : throw Return(value) -> caught by LoxFunction.call()

ERROR PATHS
-----------
Compile-time errors (scanner/parser/resolver):
  -> Lox.error(...) -> hadError = true
  -> runFile exits with status 65

Runtime errors:
  -> throw RuntimeError -> interpreter.interpret() catches
  -> Lox.runtimeError(error) -> hadRuntimeError = true
  -> runFile exits with status 70

KEY DATA STRUCTURES
-------------------
  Token        : type, lexeme, literal, line
  Expr / Stmt  : AST nodes generated by com.tool.GenerateAst
  Environment  : HashMap of values + enclosing reference
  locals       : Map<Expr, Integer> in Interpreter (filled by Resolver)
  LoxFunction  : closure + declaration + isInit flag
  LoxClass     : name + superclass + methods map
  LoxInstance  : LoxClass + fields map
